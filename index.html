HOW IT WORKS

From brief to paid in four steps.

No back-and-forth. No scope creep. No chasing invoices.

01 — SCOPE
Describe the work in plain language.
GPT-4o decomposes your brief into discrete, deliverable tasks with explicit spec constraints. JSON fields, file formats, pixel counts, sample rates — the exact rules for "done".
02 — FUND
Lock funds before work begins.
When a contributor claims a task, Stripe reserves the payout instantly via payment intent. Contributors know the money is there before they start. Hosts know no one works for phantom promises.
03 — VERIFY
AI checks delivery before human review.
Submissions run through an automated verifier that checks file headers, resolution, duration, and content against the task spec. Garbage in, bounced out automatically.
04 — RELEASE
Host approves or 48h auto-release.
Host has 48 hours to review. If they don't act, funds auto-release to the contributor. Every decision is immutably logged. No arguing about "done".

HOW IT'S BUILT

The technical stack.

A complete, production-grade system — no toy demos, no placeholder code.

Frontend
Vite SPA — vanilla TypeScript, string-returning components, Swiss-brutalist dark UI
Auth
Supabase Auth — magic link + Google OAuth with role-gated redirects (host / contributor)
Database
PostgreSQL via Supabase — RLS policies, pg_cron for claim expiry + auto-release
AI Scoping
GPT-4o via edge function decomposes natural-language briefs into JSON task specs
Payments
Stripe Connect — payment intents with manual capture for 24h escrow lock on claim
Realtime
Y.js + y-webrtc for live collaborative review sessions with cursor presence
Verification
Cloudflare Worker validates file MIME, size, and content before entering review queue
Storage
Supabase Storage — presigned URLs for direct upload, private buckets per project
